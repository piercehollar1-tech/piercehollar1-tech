## What I'm building

### AuraDesk

An AI workflow automation venture for solo wellness practitioners, such as independent massage therapists and small clinics. These are people who are great at their craft but lose hours every week to admin, and lose revenue to small gaps like missed calls and clients who quietly stop booking. AuraDesk runs those repetitive touchpoints for them so they can stay focused on clients.

The four core workflows I designed:

- **Missed call response** so an unanswered call turns into a text and a booking link instead of a lost client.
- **Review requests** that go out automatically after an appointment, at the moment a client is happiest.
- **Client reactivation** that reaches out to people who haven't booked in a while before they drift away for good.
- **Lead nurture** that follows up with new inquiries so they don't go cold.

How I'm building it:

- **Discovery and validation:** I validated the niche with market research across several sources before writing any code, then defined the product, the pricing, and the workflows above.
- **Compliance first:** I researched the legal and compliance landscape up front, including HIPAA boundaries, TCPA and SMS consent rules, and Indiana licensing, so the service is built responsibly rather than retrofitted later.
- **The stack:** the automation layer runs on n8n and the Claude API, with a working internal dashboard prototype deployed on Vercel for managing clients and workflows.
- **Where it is now:** pre-launch, and I'm preparing a first pilot with a local massage therapy practice to test it against real day to day operations.

### A personal AI operating layer

The environment I run my own work in, built on Claude Code. It started as a way to stop losing context between work sessions and grew into a small system:

- A **file based memory system** that keeps project context and lessons across sessions and surfaces the relevant ones automatically, so I'm not re-explaining the same things every day.
- A **staged skill pipeline with review gates**, adapted from open source patterns, for repeatable multi step work that pauses for a human checkpoint between stages instead of running unattended.
- A **verification habit** I added after an early silent failure: confirm that every component the system actually loads, not just that the files exist.

It's a working example of how I think about AI tooling, that it should be reliable, reviewable, and earn trust before you hand it real work.
